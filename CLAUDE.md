## REQUIRED READING BEFORE EDITING

**Before making any change to a `.Rmd` file, read [STYLE-GUIDE.md](./STYLE-GUIDE.md) in full.**

The style guide is binding. It locks down math notation (symbol assignments, sub/superscript order, `\text{}` vs `\mathrm{}` conventions, distribution notation, hat/bar/tilde meanings, units), voice (first-person plural, hedging, citation patterns), and structural conventions. Math fidelity is non-negotiable — do not improvise on notation. When in doubt, match the surrounding section verbatim and ask the maintainer before introducing a new symbol.

Key hard rules from the style guide:
- Do not invent math symbols for new parameters — propose and pause for approval.
- Do not re-derive an equation beyond what MOSAIC-pkg or laser-cholera implements.
- Do not change an existing documented symbol.
- Use inline Markdown hyperlinks for citations, never `[@key]` BibTeX form.
- Use `\varepsilon` (not `\epsilon`) for natural-immunity waning.
- Units in negative-exponent form (`\text{day}^{-1}`), never slash form.

## Documentation Repository Structure

This repository contains the documentation website for the MOSAIC cholera transmission modeling framework, built using R Bookdown and published to GitHub Pages.

## Key Development Commands

### Building Documentation
```bash
# Build the bookdown site
Rscript -e "bookdown::render_book('index.Rmd')"

# Preview locally
Rscript -e "bookdown::serve_book()"

# Clean build artifacts
Rscript -e "bookdown::clean_book()"
```

### Site Management
```bash
# Deploy to GitHub Pages (if configured)
# The built site in docs/ is automatically deployed

# Check links
Rscript -e "bookdown::check_book()"
```
